Mac
=====

在 Mac 的终端（Terminal）中，想要确认指定路径下每个文件夹的大小，最常用且最强大的工具是 `du` 命令（Disk Usage）。

以下是几种常用的命令组合，你可以根据需求选择：

### 1. 查看当前目录下每个文件夹的大小 (最常用)

如果你已经通过 `cd` 进入了目标目录，输入以下命令：

```bash
du -sh */

```

* **`-s` (summary):** 只显示该目录的总计大小，不列出子目录下的每一个文件。
* **`-h` (human-readable):** 以易读的格式显示（如 10MB, 2GB），而不是字节数。
* **`*/`:** 仅针对文件夹进行统计。

---

### 2. 查看指定路径下所有文件夹并排序

如果你想知道哪个文件夹占用的空间最大，可以结合 `sort` 命令：

```bash
du -sh /path/to/directory/* | sort -rh

```

* **`/path/to/directory/*`:** 替换为你真实的文件夹路径。
* **`sort -rh`:** 按数字反向排序（大的在前），并能识别 `h` 参数产生的单位。

---

### 3. 限制显示的层级深度

如果一个文件夹里有太多的子文件夹，你只想看“第一层”的大小，可以使用 `-d` 参数：

```bash
du -sh -d 1 /path/to/directory

```

* **`-d 1` (depth):** 指定统计深度为 1 层。这会显示该路径下所有一级子目录的大小总和。

---

### 4. 包含隐藏文件夹

默认情况下，`*` 可能不会匹配以 `.` 开头的隐藏文件夹。如果你需要包含它们，可以使用：

```bash
du -sh /path/to/directory/.* /path/to/directory/*

```

---

### 💡 进阶建议：可视化工具

如果你觉得命令行查看不够直观，Mac 上有一些非常经典的工具可以更清晰地展示空间占用：

* **NCurses Disk Usage (`ncdu`):** 这是一个基于终端的交互式工具。如果你安装了 Homebrew，可以通过 `brew install ncdu`
  安装。它比原生的 `du` 快且支持通过键盘上下翻阅。
* **GrandPerspective / DaisyDisk:** 这些是带有图形界面的 App，通过方块的大小直观展示哪些文件占用了磁盘。

#### 示例

```shell
ncdu $HOME
```
